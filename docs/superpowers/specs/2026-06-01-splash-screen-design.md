# Splash Screen — "The Peppermint Showdown" Design

**Status:** approved in brainstorm 2026-06-01; pending spec review.

**Goal:** A cinematic candy-Western splash that introduces the cast and the hero
(a cowboy on the red-and-white peppermint stallion), set to *Peppermint Rodeo*,
resolving into the interactive 2D title screen. A premium first impression that is
never in the way — always skippable, and a 6-second version on repeat launches.

---

## Locked decisions (from brainstorm)

- **Form:** hybrid — a baked cinematic intro that resolves into a living 2D sprite title.
- **Title composition:** Western showdown (hero center on the reared stallion;
  Rustler + Slippery Pete loom opposite; canard + Professor Humbug flank low).
- **Art style:** hybrid — cover-art-quality illustrated background + hero showpiece,
  with the established cast (Humbug, Rustler, Pete, canard) composited in and
  colour-graded to match.
- **Hero art source:** Nano Banana Pro (Gemini 3 Pro Image, latest), conditioned on
  **two reference images — our cowboy + the Peppermint Rodeo cover horse.**
- **Two intros:** Full (~45–50 s, first launch) and Short (~6 s, subsequent launches);
  **both tap-to-skip.**

---

## Production pipeline

- **Key frames → NB Pro.** Every shot's start/end frame is generated by NB Pro from the
  two reference images, so the stallion + rider stay consistent. Frames **chain**
  (each shot's end frame = the next shot's start frame) for seamless cuts.
- **Motion → Veo 3.1** image→video per shot, via `tools/veo_render.sh` (extended with a
  `--last-image` flag for true start+end-frame interpolation). **Veo clips carry
  SFX/ambience only — NO music** (so Veo never imitates the track).
- **Music → the real *Peppermint Rodeo* track**, layered starting at **Shot 7** (where the
  player wanted "the music starts"). The track is owned by the `MusicPlayer` autoload and
  continues seamlessly into the 2D title and the level selector (never restarted).
  Mechanism: the cinematic cues `MusicPlayer.play_splash()` at the Shot-7 beat, so the
  track starts mid-cinematic. The existing `main_menu._ready` → `play_splash()` call stays
  as-is — the same-stream guard means it will NOT restart a track the cinematic already
  started, and it cleanly starts the track if the player skipped before Shot 7.
- **Assembly:** Veo clips concatenated, the track mixed in from Shot 7 → a single OGV
  (Godot-native), played via `VideoStreamPlayer` (mobile-safe — same path as the cowboy
  idle clips; no custom shaders).

## Short intro — 6 s, subsequent launches (proposal)

Open directly on the money shot: the reared peppermint stallion silhouetted against the
lollipop sun, cowboy's hat raised — one held beat with a whinny and the *Peppermint Rodeo*
sting — then match-cut to the 2D title. Reuses Shot 8's end frame + a single short Veo clip.
Tap-to-skip. (This is the repeat-launch experience; the full cinematic plays only when the
full-intro flag is unset.)

---

## 2D title screen (after the cinematic)

- **Showdown composition** as layered sprites: hero center on the reared horse, Rustler +
  Pete looming opposite, canard + Humbug flanking, candy-desert foreground.
- **"NOT THIS AGAIN" in the Rye font**, stamping in with a candy-wrapper bounce.
- **PLAY** button + a new **wooden settings sprocket** (gear) beside it.
- Hearts + taglines ("no ads / no pay-to-win"), as today.
- **Living idle loop:** tween-based puppet motion (horse paws/bobs, hero shifts, canard
  blinks, Humbug taps cane, villains menace, subtle parallax + dust). NO custom shaders.
- *Peppermint Rodeo* keeps playing (owned by `MusicPlayer`), continuing into the selector.

### Settings sprocket
Wooden gear sprite next to PLAY → opens a small settings panel. **v1 contents: music
volume slider + mute** (ties into the 50%-volume question; `MusicPlayer.set_music_linear`
already exists). The panel can grow later (SFX volume, credits, reset).

---

## Run logic

A persisted flag (`GameState` / save file), e.g. `full_intro_seen`:
- unset → play the **full** cinematic, then set the flag.
- set → play the **short** 6 s intro.
- Both are **tap-to-skip** → 2D title. A debug-menu reset clears the flag for testing.
- Decision to confirm at review: full intro **once ever** (recommended) vs **once per app session**.

---

## Technical implementation (mobile-safe)

- **Cinematic:** a `splash` scene (new, set as `run/main_scene`) with a `VideoStreamPlayer`
  playing the assembled OGV; on finish or tap → `change_scene_to_file` to `main_menu`
  (the 2D title). Music cued via `MusicPlayer` at the Shot-7 beat and on skip.
- **2D title:** recompose the existing `main_menu.tscn` (already holds cowboy / Humbug /
  Pete / title / PLAY) + add horse, canard, illustrated background, sprocket; animate with
  tweens (popsicle-stick-puppet style). Background plate generated via Imagen (no people).
- **No custom spatial/canvas shaders** anywhere (they white-rect on the device).

---

## Tooling to build

- `tools/nb_pro_render.py` — Gemini 3 Pro Image (Nano Banana Pro) via Vertex AI:
  prompt + **multiple reference images** → PNG. **Verify access first** (model id + auth)
  before generating the full frame set.
- `tools/veo_render.sh` — add `--last-image PATH` (Veo `lastFrame` interpolation) so a shot
  can be pinned between a start and an end frame.
